Báo cáo tổng kết

Mục tiêu sự kiện

FCAJ Technical Sharing Meeting được tổ chức nhằm chia sẻ kiến thức thực tế về Cloud Computing, Trí tuệ nhân tạo (AI), An toàn thông tin, Container, Hệ thống Multiplayer và định hướng phát triển nghề nghiệp. Sự kiện gồm nhiều bài trình bày kỹ thuật từ các thành viên FCAJ và diễn giả khách mời, giúp người tham dự học hỏi từ các dự án thực tế và kinh nghiệm trong ngành.

Các phiên chia sẻ kỹ thuật

  • Hệ thống phát hiện xâm nhập mạng (NIDS) sử dụng Machine Learning trên AWS
  • Docker – Công nghệ Containerization
  • Multiplayer trên Cloud với AWS WebSockets
  • Xây dựng GraphRAG bằng Amazon Bedrock và Amazon Neptune
  • Nghệ thuật làm việc nhóm hiệu quả
  • Hành trình từ IT Helpdesk đến Senior System Administrator

Nội dung nổi bật

AWS WAF + Machine Learning để phát hiện tấn công mạng

  • Khả năng và giới hạn của AWS WAF
  • So sánh Rule-based và Machine Learning
  • Xây dựng hệ thống NIDS bằng Machine Learning
  • Dashboard giám sát thời gian thực
  • Triển khai giải pháp bảo mật trên AWS
  • Kết hợp AWS WAF với NIDS để tăng khả năng phát hiện tấn công

Docker Containerization

  • So sánh Virtualization và Containerization
  • Kiến trúc Docker
  • Docker Images và Containers
  • Minh họa Docker trong thực tế
  • Lợi ích của việc triển khai ứng dụng bằng Container

Multiplayer trên Cloud

  • Các khái niệm về Multiplayer Networking
  • Kiến trúc WebSocket
  • API Gateway WebSocket
  • Tích hợp AWS Lambda
  • DynamoDB để quản lý kết nối
  • Demo ứng dụng Multiplayer thời gian thực

GraphRAG với Amazon Bedrock & Neptune

  • Giới thiệu Retrieval-Augmented Generation (RAG)
  • Khái niệm GraphRAG
  • Amazon Bedrock Knowledge Bases
  • Amazon Neptune Analytics
  • Ứng dụng AI khai thác mối quan hệ dữ liệu

Làm việc nhóm hiệu quả

  • Bốn nguyên tắc vàng trong làm việc nhóm
  • Giao tiếp và phối hợp
  • Trách nhiệm cá nhân
  • Công cụ cộng tác số

Hành trình nghề nghiệp

  • Từ IT Helpdesk đến System Administrator
  • Tư duy Cloud
  • Văn hóa DevOps
  • Lộ trình phát triển nghề nghiệp
  • Học tập thông qua các dự án thực tế

Kiến thức đạt được

Công nghệ Cloud

  • Hiểu cách kết hợp các dịch vụ AWS để xây dựng ứng dụng Cloud an toàn và có khả năng mở rộng.
  • Hiểu các kiến trúc thực tế sử dụng API Gateway, Lambda, DynamoDB, Amazon Bedrock, Neptune và AWS WAF.

An toàn thông tin

  • Machine Learning giúp nâng cao khả năng phát hiện các hành vi bất thường mà các hệ thống Rule-based có thể bỏ sót.
  • Kết hợp AWS WAF với NIDS giúp tăng cường khả năng bảo vệ trước các cuộc tấn công hiện đại.

Kỹ thuật phần mềm

  • Docker giúp đơn giản hóa việc triển khai ứng dụng và tăng tính nhất quán trong phát triển.
  • GraphRAG mở rộng mô hình RAG truyền thống bằng cách khai thác mối quan hệ giữa các dữ liệu.
  • WebSocket hỗ trợ giao tiếp thời gian thực cho các ứng dụng Multiplayer.

Phát triển nghề nghiệp

  • Làm việc nhóm hiệu quả phụ thuộc vào giao tiếp, mục tiêu chung, trách nhiệm và sự hợp tác.
  • Thực hành trên các dự án thực tế và học tập liên tục là yếu tố quan trọng để phát triển sự nghiệp trong lĩnh vực Cloud và DevOps.

Trải nghiệm cá nhân

Buổi FCAJ Technical Sharing Meeting mang đến cho tôi nhiều kiến thức thực tế về các chủ đề Cloud Computing trong cùng một sự kiện. Khác với những buổi học lý thuyết, mỗi bài trình bày đều tập trung vào các dự án thực tế và cách triển khai trong doanh nghiệp.

Tôi đặc biệt ấn tượng với các phiên chia sẻ về AWS WAF kết hợp Machine Learning và GraphRAG vì chúng cho thấy cách các dịch vụ AWS hiện đại có thể kết hợp với AI để giải quyết các bài toán thực tế. Phần trình bày về Docker cũng giúp tôi hiểu rõ hơn về Containerization – một công nghệ rất phổ biến trong các ứng dụng Cloud-Native.

Ngoài kiến thức chuyên môn, tôi còn có cơ hội trao đổi với các thành viên khác, chia sẻ kinh nghiệm học tập và hiểu thêm về xu hướng công nghệ hiện nay.

Bài học rút ra

  • Các giải pháp bảo mật nên kết hợp giữa phương pháp truyền thống và Machine Learning.
  • Docker đã trở thành công nghệ quan trọng trong việc triển khai phần mềm hiện đại.
  • GraphRAG giúp nâng cao khả năng truy xuất tri thức bằng cách khai thác mối quan hệ giữa dữ liệu.
  • Các dự án Cloud thực tế đòi hỏi khả năng làm việc nhóm và giao tiếp hiệu quả.
  • Tham gia cộng đồng kỹ thuật giúp đẩy nhanh quá trình học tập và phát triển nghề nghiệp.
